# Livraison Site (SnackExpress)

## Description
This is a static single‑page‑app (SPA) allowing users to order snacks. The checkout form sends data to **FormSubmit.co**, which delivers the order via email to `LordainzAF4@gmail.com`.

## Features
- Cart stored in `localStorage`
- Dynamic receipt with order number
- WhatsApp button that pre‑fills the delivery phone number
- Email subject is formatted as `🚨 Commande N° <orderId> — <name>`

## Required files for deployment
- `index.html` (already present)
- `css/`, `js/`, `img/` folders with your assets
- **`_redirects`** – redirects all routes to `index.html` (needed for SPA on GitHub Pages / Render)
- **`404.html`** – fallback page for GitHub Pages
- **`.gitignore`** – ignore node modules, build output, logs, etc.

## Deployment (GitHub Pages)
1. Create a repository on GitHub.
2. Push the entire `livraison_site_com` folder.
3. In the repo Settings → Pages, enable **Source: `main` branch**.
4. After the site builds, GitHub will provide a URL like `https://<user>.github.io/<repo>/`.
5. Open the URL, submit a test order, click the activation email from FormSubmit, then confirm that order emails arrive.

## Deployment (Render – static site)
1. Create a new *Static Site* on Render and connect it to this GitHub repo.
2. Set **Build Command** empty and **Publish Directory** to `.`.
3. Render will automatically use the `_redirects` file for SPA routing.

## Important notes
- **FormSubmit** only works on publicly accessible URLs (not `file://`).
- After the first activation email you will receive all subsequent order emails automatically.
- Ensure the email address in `index.html` (line ~2130) matches `LordainzAF4@gmail.com`.
